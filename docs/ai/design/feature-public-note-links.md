# Public Note Links Design

## Architecture Overview

The feature adds shared core services for public note links, a Supabase persistence/API layer, web UI for creating and viewing links, and mobile owner UI for creating/sharing the same public web links. The public viewer route is intentionally separate from `NotesShell` so recipients never load private navigation, note lists, search, or editing controls.

```mermaid
graph TD
  Owner[Authenticated note owner] --> WebMenu[Web note three-dot menu]
  Owner --> MobileMenu[Mobile note options menu]
  WebMenu --> ShareDialog[Share note dialog]
  MobileMenu --> MobileShareSheet[Mobile share note sheet]
  ShareDialog --> CoreShare[Core PublicNoteShareService]
  MobileShareSheet --> CoreShare
  CoreShare --> Supabase[Supabase client]
  Supabase --> ShareLinks[(note_share_links)]
  ShareLinks --> Notes[(notes)]
  Recipient[Anyone with link] --> PublicRoute[/share/?token=...]
  PublicRoute --> PublicCore[Core public note reader]
  PublicCore --> PublicRpc[get_public_note_by_token RPC]
  PublicRpc --> ShareLinks
  PublicRpc --> Notes
  PublicRoute --> ReadOnlyPage[Dedicated read-only web UI]
```

## Data Models

### `note_share_links`

- `id uuid primary key`
- `note_id uuid not null references notes(id) on delete cascade`
- `user_id uuid not null references auth.users(id) on delete cascade`
- `token text not null unique`
- `permission text not null default 'view'`
- `is_active boolean not null default true`
- `created_at timestamptz not null default now()`
- `updated_at timestamptz not null default now()`

Indexes:

- Unique active view link per note for MVP: partial unique index on `(note_id)` where `permission = 'view' and is_active`.
- Lookup index on `token`.
- Owner index on `(user_id, note_id)`.

Defaults and constraints:

- `token` is generated by the database from `gen_random_uuid()` with hyphens removed, keeping token creation platform-neutral for web and future mobile.
- `permission` is constrained to `view` for the MVP.
- Insert/update RLS policies validate that `auth.uid()` owns both the share-link row and the referenced note.

### Core Types

- `PublicNoteSharePermission = 'view'`
- `PublicNoteShareLink`: owner-facing link metadata.
- `PublicNote`: recipient-facing note projection containing `title`, `description`, `tags`, and date metadata.

## API Design

Core service methods:

- `getOrCreateViewLink(noteId, userId): Promise<PublicNoteShareLink>`
- `getPublicNoteByToken(token): Promise<PublicNote | null>`
- `buildPublicNoteUrl(origin, token): string`

Supabase operations:

- Authenticated owners create/reuse links through `PublicNoteShareService` using normal Supabase client auth and RLS.
- Public viewers fetch the shared projection through a security-definer RPC `get_public_note_by_token(token text)` that returns only active link rows joined to one note projection.

Authorization:

- Owners can create/read their own `note_share_links`.
- Owners can only create links for notes they own.
- Public viewers cannot query `notes` directly.
- RPC runs with a fixed `search_path`, accepts only a token, and does not expose `user_id`, private note IDs beyond the returned public projection, or other notes.

## Component Breakdown

- `core/services/publicNoteShare.ts`: shared service and URL builder.
- `core/tests/unit/core-services-publicNoteShare.test.ts`: unit tests for service behavior.
- `supabase/migrations/*_add_public_note_links.sql`: table, RLS, indexes, RPC.
- `ui/web/components/features/notes/ShareNoteDialog.tsx`: owner popup with permission row, generated link, copy action, loading/error states.
- Existing note menu components: add "Share note" menu item with share icon.
- `app/share/page.tsx`: static-export-compatible public web route with `noindex` metadata.
- `ui/web/components/features/public/PublicSharePageClient.tsx`: reads the `token` query parameter, fetches one note by token, and renders loading/not-found states.
- `ui/web/components/features/public/PublicPageHeader.tsx`: sticky lightweight public header with EverFreeNote identity and theme toggle only.
- `ui/web/components/features/public/PublicNotePage.tsx`: presentational read-only layout with sanitized content and non-clickable tag badges.
- `ui/mobile/components/ShareNoteDialog.tsx`: native bottom sheet that generates/reuses the view link and exposes the public URL plus native share action.
- `ui/mobile/components/NoteIndexMenu.tsx`: mobile note options sheet includes "Share note" alongside AI index actions.
- `ui/mobile/adapters/config.ts`: resolves the public web origin for mobile URL building.

## Design Decisions

- Database-generated token-based links avoid exposing note IDs as authorization credentials and avoid duplicating token generation logic across web/mobile.
- A dedicated public route avoids accidental leakage through the authenticated app shell.
- A core service keeps creation/retrieval contracts reusable for mobile while the first UI remains web-only.
- The MVP reuses an existing active view link for the same note to avoid duplicate share URLs.
- `is_active` is included now so revocation can be added later without a breaking migration.
- Public page uses `noindex` metadata as a privacy-friendly default.
- Static export compatibility requires the share token to live in the query string (`/share/?token=...`) instead of a dynamic app route segment.
- Public viewers can switch light/dark theme using the existing web `ThemeProvider`, without exposing private navigation, search, note lists, or editing controls.
- Mobile generates the same `/share/?token=...` public web URLs. `getPublicWebOrigin()` validates explicit `http`/`https` mobile config when present, otherwise infers an origin from the configured editor WebView URL.

## Non-Functional Requirements

- Security: tokens must be high entropy, public RPC must return exactly one active note projection, and public rendering must sanitize rich content.
- Performance: token lookup should be indexed and perform one bounded query/RPC call.
- Accessibility: share dialog needs clear labels, keyboard focus handling, copy feedback, and no color-only status.
- Accessibility: public theme switching needs an accessible icon button label and should be available on public loading/error/not-found states as well as the note view.
- Reliability: failed link generation should leave existing note state unchanged and show retryable error UI.
- Mobile readiness: no browser globals inside core service; URL origin is injected by web/mobile adapters.
